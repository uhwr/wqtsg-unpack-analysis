# 我去图书馆小程序解包分析

## 项目简介

本仓库用于整理“我去图书馆”相关微信小程序与管理端前端的解包结果、还原代码样本和静态分析文档，方便后续持续补充接口观察、页面逻辑梳理与行为链路研究。

当前仓库的重点不在于重新开发业务，而在于保留分析现场，包括：

- 小程序解包后的还原目录
- 管理端前端代码样本
- 蓝牙签到相关逻辑分析
- 管理端页面、接口、数据流与权限点梳理
- 已完成的阶段性分析文档

## 仓库定位

这个仓库是一个“资料归档 + 分析记录”仓库，主要服务于以下场景：

- 回看某次解包后的原始目录结构
- 对照分析文档定位代码位置
- 继续补充接口、页面和调用链分析
- 作为后续审计、研究、复核的基础材料

仓库内保留的内容以“可用于分析”和“可用于复现阅读上下文”为原则，不保留无关脚本、备份目录和临时工具文件。

## 目录结构

```text
.
├─ README.md
├─ .gitignore
├─ wx3b9352e6b254ed2b_unpacked_还原模块/
├─ 我去图书馆管理端/
└─ 相关文档/
```

各目录说明如下：

- `wx3b9352e6b254ed2b_unpacked_还原模块/`
  保存小程序解包后的还原结果，可用于查看页面、组件、工具函数、配置项和打包产物中的业务逻辑。

- `我去图书馆管理端/`
  保存管理端前端代码样本，便于分析后台页面结构、接口调用方式、状态处理和页面功能分布。

- `相关文档/`
  保存对解包结果和管理端代码的阶段性分析结论，是仓库中最适合快速阅读全局结论的部分。

## 文档索引

`相关文档/` 目录中当前主要包括：

## 关键代码路径索引

下面这些路径适合作为后续继续分析时的常用入口：

- `wx3b9352e6b254ed2b_unpacked_还原模块/app-service.js`

- `wx3b9352e6b254ed2b_unpacked_还原模块/config.js`
  常用于查看后端地址、接口配置或环境相关常量。

- `wx3b9352e6b254ed2b_unpacked_还原模块/pages/blue/`
  蓝牙签到相关页面目录，适合优先排查签到流程、页面跳转和字段使用。

- `wx3b9352e6b254ed2b_unpacked_还原模块/pages/free/`
  可用于结合文档继续梳理座位、管理或其他业务页面逻辑。

- `wx3b9352e6b254ed2b_unpacked_还原模块/utils/http.js`
  请求封装入口，适合用于反查接口调用方式、统一参数处理和返回值格式。

- `wx3b9352e6b254ed2b_unpacked_还原模块/service/`
  服务层目录，可用于追踪业务请求封装和数据处理逻辑。

- `我去图书馆管理端/app-service.js`
  管理端打包后的主业务文件，适合从整体入口回溯页面与模块。

- `我去图书馆管理端/pages/`
  管理端页面集合，可按页面名逐步建立功能地图。

- `我去图书馆管理端/components/`
  管理端公共组件目录，适合排查复用弹窗、选择器、布局与编辑器组件。

如果后面要继续补文档，建议在分析结论里直接引用这些相对路径，方便仓库内交叉定位。

## 建议阅读顺序

如果第一次接触这个仓库，建议按下面顺序阅读：

1. 先看 `相关文档/我去图书馆管理端_初步分析.md`，快速建立整体认识。
2. 再看 `相关文档/我去图书馆管理端_深入分析.md`，进入接口与页面细节。
3. 接着看 `相关文档/蓝牙签到与验证码校验总结.md`，聚焦签到链路。
4. 最后结合 `wx3b9352e6b254ed2b_unpacked_还原模块/` 和 `我去图书馆管理端/` 对照原始代码定位具体实现。
